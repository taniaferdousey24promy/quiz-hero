Ques-1: What is the difference among map( ), filter( ), forEach( ), find( )?



`map()` `filter()` `forEach()` `find()`
map() is used to transform each element of an array into a new value or object and create a new array with those transformed values. filter() is used to select a subset of elements from an array based on a specified condition and create a new array with those selected elements. forEach() is used to perform a certain action or operation on each element of an array. find() is used to retrieve the first element from an array that satisfies a specified condition.
map() returns a new array with the same length as the original array, where each element is the result of applying a function to the corresponding element of the original array. filter() returns a new array containing only the elements of the original array that pass a certain condition specified by a function. forEach() does not return anything; it is used to perform an action on each element of an array. find() returns the first element of the original array that satisfies a condition specified by a function. If no element satisfies the condition, undefined is returned.
map() iterates over all elements of the array filter() iterates over all elements of the array forEach() iterates over all elements of the array find() iterates over all elements of the array until a match is found

Ques-3: What is Event Loop and how does it work ?

Answer: The event loop has simple job to monitor the Call Stack and the Callback Queue. It is a process that waits for the Call Stack to be clear before pushing callbacks from the Task Queue to the Call Stack. Once the Stack is clear, the event loop triggers and checks the Task Queue for available callbacks.

  1. When an event or task is registered, it is added to a queue known as the "task queue". The Event Loop then continuously checks this queue for pending tasks. If there are tasks waiting to be executed, the Event Loop takes each task from the queue one by one, and executes it.
  2. Once a task is executed, the Event Loop checks if any other tasks have been added to the queue. If there are, it executes them in the same way, otherwise it waits for new tasks to be added to the queue before checking it again.

  3. This process of constantly checking the task queue for pending tasks, executing them one by one, and then waiting for new tasks to be added is what allows asynchronous programs to be non-blocking, and allows them to handle multiple operations concurrently.
